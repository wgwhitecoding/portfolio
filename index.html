CanDo-Kansan Board
Live Demo Source Code
Date Created: July 15, 2024
Project Overview:

The Kanban Board Project is a comprehensive task management application developed as the capstone of my Code Institute Full-Stack Developer course. This project demonstrates my ability to architect and implement complex web applications using Django as the backend framework, alongside Bootstrap for responsive design. The application features full CRUD (Create, Read, Update, Delete) functionality, empowering users to manage tasks, set priorities, and track project progress seamlessly.

Building this project honed my understanding of core Django concepts like model-view-template (MVT) architecture, user authentication, and database interactions using Django ORM. As my first solo Django project, this experience reinforced my skills in backend development, while also enhancing my ability to create intuitive, user-friendly interfaces that prioritize functionality and aesthetic coherence.

Features:
  • Create and manage tasks
  • Task tracking across different stages
  • Responsive design with light and dark modes
  • Kanban board with drag-and-drop functionality
  • Task prioritization with due dates and notifications
  • User authentication (login/register)
  • Profile editing and password change
  • Customizable background (personal image upload)
  • Search functionality for tasks
  • CRUD functionality for tasks and columns
Tech Stack (Technologies Used):
Django & Python:

Django, a high-level Python web framework, handled the entire backend. It streamlined the development of features like user authentication, profile management, and task CRUD operations using its built-in models and views system. Python's simplicity allowed for clean, scalable code, while Django ORM enabled secure, efficient database interactions. Django's robust form handling ensured that features like task creation, editing, and deletions were processed securely. Custom views were used to handle the logic for Kanban functionality, while templates integrated front-end components dynamically.

HTML:

HTML was used to structure the application’s pages, integrating Django’s template language to render dynamic content based on user interactions.

CSS:

CSS defined the styling for the entire application, leveraging Bootstrap’s framework for responsive layouts and custom styles for specific design elements like the Kanban board.

JavaScript:

JavaScript added interactivity, such as drag-and-drop for tasks, dynamic updates, and enhancing the user experience with responsive elements and animations.

Bootstrap:

Bootstrap provided responsive, pre-styled components like forms, buttons, and grids, ensuring that the app functioned seamlessly across devices.

PostgreSQL:

PostgreSQL, the database, stored user data, task details, and project information. Django ORM interacted with PostgreSQL to manage data efficiently and securely.

Heroku:

The application was deployed on Heroku, a platform-as-a-service (PaaS), which ensured smooth deployment and integration with PostgreSQL.

Learning Outcome:

Developing this project enhanced my skills in Django, PostgreSQL, and JavaScript. I deepened my understanding of the MVC pattern, user authentication, and CRUD functionality, as well as how to deploy a full-stack application using Heroku.

Future Enhancements:

I plan to add user analytics for tracking productivity, integration with services like Google Calendar, and real-time updates for team collaboration. Other potential improvements include task automation and enhanced customization features for users.

Who Wants to Be a Millionaire
Live Demo Source Code
Date Created: July 15, 2024
Project Overview:

The"Who Wants to Be a Millionaire" project is an interactive web-based quiz game inspired by the iconic TV show. Players are presented with a series of progressively challenging multiple-choice questions, with the option to use lifelines like 50/50 and Ask the Audience. The game tracks players' scores, providing feedback after each question. JavaScript drives the game’s logic, ensuring smooth transitions and lifeline functionality, while HTML and CSS shape the user interface. Although the game is not yet fully responsive, ongoing work aims to make it adaptable to different screen sizes, enhancing the user experience across devices..

This project reflects strong JavaScript development skills, as well as a solid understanding of front-end design principles.

Features:
  • Interactive quiz game with multiple-choice questions
  • Progressively more difficult questions as the player advances
  • Lifelines (50/50 and Ask the Audience) to assist players
  • Score tracking system that updates dynamically
  • Feedback provided after each question
  • Game logic implemented with JavaScript for smooth transitions
  • Responsive design (currently in progress)
Tech Stack (Technologies Used):
JavaScript:

JavaScript is the core of the game’s logic. It dynamically updates questions, tracks player progress, and manages lifeline features like 50/50 and Ask the Audience. By interacting with the DOM, it ensures that the user interface reacts instantly to player inputs, providing a smooth, uninterrupted experience. It also includes score tracking, difficulty progression, and real-time feedback for players without reloading the page.

CSS:

CSS was employed to style the game and enhance its visual appeal. Although not yet responsive, it includes a structured layout for the question display and user inputs. Flexbox was utilized to align buttons and ensure consistent design. Upcoming enhancements will focus on making the game mobile-friendly through responsive design techniques.

HTML:

Used to structure the game interface, including questions, answers, and lifeline buttons. HTML forms the backbone of the game’s layout, ensuring the necessary elements are presented in a clear and structured way to facilitate easy interaction for users.

Learning Outcome:

This project significantly improved my front-end development skills. I gained a deeper understanding of JavaScript, particularly in managing dynamic game states and interacting with the DOM to update the user interface in real-time. Handling user inputs and integrating lifelines provided valuable experience in event-driven programming. Moreover, the project highlighted the importance of structuring HTML and CSS for interactive experiences, teaching me how to build scalable, user-friendly interfaces. It also helped me identify areas for improvement, particularly in making the game responsive, which remains an ongoing focus.

Future Enhancements:

Future enhancements for this project include making the interface fully responsive, so the game works seamlessly across all screen sizes and devices. Additional improvements may include introducing new lifelines (such as Phone a Friend), expanding the question database, and implementing sound effects and animations to enhance engagement. There is also potential for adding a user authentication system, enabling players to save their progress, track high scores, and compete on leaderboards. These enhancements will further improve user experience and make the game more interactive and enjoyable.

Project Overview:

The Crew Weather project was created during a hackathon, emphasizing group collaboration and the effective use of APIs. It’s a weather forecast application that fetches real-time data based on the user’s location. The app provides a seamless user experience by integrating an external weather API, offering live updates and weather conditions in a clean, intuitive interface. The team’s focus was to create a fully responsive application that could be accessed from any device, making weather forecasting both easy and accurate. This project showcases the ability to work in a fast-paced, team-oriented environment while utilizing modern web technologies.

This project reflects strong JavaScript development skills, as well as a solid understanding of API intergration principles.

Features:
  • Real-time weather updates based on user location
  • Search functionality for weather conditions by city
  • Integration with an external API for fetching live data
  • Responsive design, adaptable to desktop and mobile devices
  • Clean and minimalistic interface for ease of use
Tech Stack (Technologies Used):
API Integration:

The app integrates with a third-party weather API to retrieve up-to-the-minute weather conditions. JavaScript makes API calls, processes the returned data, and dynamically updates the DOM to reflect the live weather updates for users.

JavaScript:

JavaScript drives the core functionality of the app by handling asynchronous requests to the external weather API. It manages dynamic updates for weather conditions based on user location or city input. JavaScript is responsible for the smooth interaction between the user and the real-time data..

CSS:

CSS was implemented to create a visually appealing, responsive design that adjusts to different screen sizes. Flexbox and media queries were used extensively to maintain layout consistency on both mobile and desktop devices..

HTML:

Used to structure the weather application and present weather data in an organized manner. HTML elements include forms for city search input and containers for displaying current weather conditions and updates.

Learning Outcome:

This project helped me sharpen my API integration skills, particularly in handling asynchronous JavaScript operations and parsing external data. Working within a team environment during the hackathon reinforced collaboration skills and time management under tight deadlines. I also gained more experience in designing responsive layouts and ensuring that the application works seamlessly across different devices.

Future Enhancements:

To enhance the user experience, future improvements could include adding hourly and weekly weather forecasts, notifications for severe weather conditions, and more detailed weather data such as humidity and wind speed. Visual enhancements such as icons and weather animations will also be incorporated to make the interface more engaging.
